    • Introduction

    Welcome to the Kalay Device Management API! You can use this API for registering your device with Kalay Device Management (a.k.a. DM) Server, updating data, and also retrieve data from the Kalay Data Server.

    Currently we have API support for Android in Java, and are on our way delivering an iOS version.

    Working with Kalay DM Server

    Wakaama mWakaama = new Wakaama();
mWakaama.init();

    In order to connect your device to the Kalay DM Server, you have to initialize a Wakaama object by running init.

    Client Configuration

    
int lifetime = 30; // Unit: second
String endpoint_name = "Device001";
String endpoint_port = "56830";
String server_name = "52.42.150.111";
String server_port = "5683";

setLifetime( lifetime );
setName( endpoint_name );
setPort( endpoint_port );
setServerName( server_name );
setServerPort( server_port );
setIPv4();

    After running the init step, you may explicitly call one of the following methods before you register to and start working with the DM Server --

    1. setLifetime: This method is used for setting up the lifetime for device registration update interval. The input should be integer and the unit is second. The default value is 30 and it should be set no greater than this value when using IPv4 in order to keep the connection alive.
    2. setName: This method should be used for setting up the endpoint name of your device. This name will be later used when a data retriever or any other observers try to observe your device. In other words, the name will be the ID of your device and it should be a string that only contain simple alphanumeric characters.
    3. setPort: This method should be used for setting up the port of your device. Normally it can be leaved unchanged.
    4. setServerName: This method should be used for setting up the host name of the server you are connecting to. The current DM Server only supports IPv4. The input should be a string that contains full IP or domain name.
    5. setServerPort: This method should be used for setting up the port of the server you are connecting to. The default port is 5683. It is the default port used by CoAP, the Transport-Layer protocol for our communication system.
    6. setIPv4 This method is used for switching the network connection to IPv4 connection. This method doesn't take any input. Since IPv4 is the default for our service, there's no need for explicit call for this method.
    String endpoint_name = getName();
String endpoint_port = getPort();
String server_name = getServerName();
String server_port = getServerPort();
boolean isIPv4 = getIPv4();

    In contrast, there are methods getting the information set by the methods above --

    1. getName: This method is used for getting the endpoint name of the device. The method returns a string.
    2. getPort: This method is used for getting the port of the device. The method returns a string.
    3. getServerName: This method is used for getting the address of the DM Server. The method returns a string.
    4. getServerPort: This method is used for getting the port of the DM Server to which the device is connected. The method returns a string.
    5. getIPv4: This method returns true if the connection is running through IPv4, and false otherwise.

    Defiderm Device Data Update

    For a connected DM Defiderm Client, there are currently 28 methods for updating the object resources. These methods are used for updating the client with the values got from a physical device. All these methods have two inputs -- the first ones are instance IDs, and the second ones the actual data.

    Instance ID is used to distinguish devices when multiple devices are connected to the app. For the case that only one device is connected, its instance ID should be 0.

    String Type

    String value;

setAccountIdValue(0, value);
setMacValue(0, value);
setSkinValue(0, value);
setUserIdValue(0, value);
setSerialNumberValue(0, value);
setVendorCodeValue(0, value);
setDeviceIdValue(0, value);
setHydrationLevelDescriptionValue(0, value);
setIdValue(0, value);
setMelaninLevelDescriptionValue(0, value);
setRednessLevelDescriptionValue(0, value);

    For the following data types, please insert string type of values. The upper bound of the length of these strings are limited to 128 characters.

    1. accountId
    2. mac
    3. skin
    4. userId
    5. serialNumber
    6. vendorCode
    7. deviceId
    8. hydrationLevelDescription
    9. id
    10. melaninLevelDescription
    11. rednessLevelDescription

    Integer Type

    long value;

    For the following data types, please insert long type of values. These two values are timestamps, and should be integers that represent seconds after Unix epoch.

    1. createdAt
    2. lastModifiedAt
    byte value;

    For the following data types, please insert byte type of values.

    1. hydrationLevel
    2. melaninLevel
    3. rednessLevel

    Data Retrieval

    The data retrieval part is currently not an intrinsic part of the library, since it is offered by the REST API from the Kalay Data Server. It can be done by a simple HTTP PUT action followed by an open Web Socket listing to the service. The data will be then retrieved whenever it is updated.

    Web Socket Connection

    String mHost = "dm.kalay.co";
int mPort = 80;
String mToken; // Please get this from the Kalay Account Management Service

HashMap<String, String> headers = new HashMap<String, String>();
headers.put("Origin", String.format("%s/%d", mHost, mPort));
headers.put("Authorization", "Bearer " + mToken);

    In order to run the data retrieval procedure, a Web Socket should be initialized with a header with two parts of information --

    1. "Origin": The content of this should be the URL of the Kalay Data Server, which is currently "dm.kalay.co:80".
    2. "Authorization": The content of this should be the string "Bearer " followed by a token got from the Kalay Account Management (a.k.a. AM) Service.
    String mHost = "dm.kalay.co";
int mPort = 80;
String mEndpoint = "Device001";
String mResource = "10000/0";

URI uri = new URI(String.format("ws://%s:%d/devices/%s/%s", mHost, mPort, mEndpoint, mResource));

    The Web Socket can be connected right at the very beginning of the entire data retrieval process. It should connect to the URI that is pointing to the endpoint name and object instance path of the desired data source.

    The current Defiderm Object Model defines the object ID as 10000. In addition, when connecting to the only instance of that object, the instance ID should be 0. In other words, 10000/0 should be the object instance path in most cases.

    In addition, the user can also choose to listen to a single resource instead of the entire object. For this use case, simply insert the full path to the resource to the URI. For instance, 10000/0/2 for observing the battery value of the first Defiderm instance. The resource IDs are listed in the end of this document.

    HTTP Observe

    String mHost = "dm.kalay.co";
int mPort = 80;
String mEndpoint = "Device001";
String mResource = "10000/0";

URL url = new URL(String.format("http://%s:%d/devices/%s/%s/observe", mHost, mPort, mEndpoint, mResource));

HttpURLConnection conn = (HttpURLConnection) url.openConnection();
conn.setRequestProperty("Authorization", "Bearer " + mToken);
conn.setDoOutput(true);
conn.setRequestMethod("PUT");

    As for observing the data, simply trigger the process by notifying the Kalay Data Server by processing an HTTP PUT to the resource path followed by "observe".

    After these two processes are run, the data retrieval process is set and the observer will get updates everytime when the client data is updated.
